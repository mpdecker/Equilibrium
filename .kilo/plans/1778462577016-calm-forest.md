# Plan: Create Android Native Version

## Goal
Generate the Capacitor Android native project for the Equilibrium app, reusing all existing web code.

## Code Reuse Strategy

Capacitor's architecture inherently reuses nearly all web code — the Android app is a thin native shell that hosts a WebView running the exact same `dist/` build as the web version:

- **Web UI (React)**: All components, pages, and UI — 100% reused in WebView
- **Audio engine (Tone.js)**: Runs in the WebView's JS context — no native audio code needed
- **Business logic**: Random generators, state management, API calls — identical across platforms
- **Capacitor plugins**: `@capacitor/app` already has a native Android counterpart auto-generated during sync
- **Native layer**: Only Capacitor's auto-generated shell (MainActivity, Gradle config) — zero custom native code required

## Steps

### 1. Build the web app
```bash
npm run build
```
Produces the `dist/` production bundle. This is the single shared codebase that Android will load in its WebView.

### 2. Add Android platform
```bash
npx cap add android
```
Creates the `android/` directory with the native shell project (Gradle build, AndroidManifest.xml, MainActivity, and Capacitor plugin wiring). Automatically includes the native side of `@capacitor/app`.

### 3. Sync web assets to Android
```bash
npx cap sync android
```
Copies `dist/` into `android/app/src/main/assets/public/` and updates Capacitor plugin references. After this step, the Android app loads and renders the web app identically.

### 4. Verify
Confirm `android/` directory exists with the Gradle project structure. Optional: open in Android Studio to build/run:
```bash
npx cap open android
```
