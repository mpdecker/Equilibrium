# Code Audit & Test Suite: Equilibrium

## Project Summary
Equilibrium is a generative ambient music web app (React 19 + Express + Tone.js + Gemini AI) for mindfulness/emotional regulation. It has **zero existing tests** and no linter beyond `tsc --noEmit`.

## Phase 2: Code Audit (Issues to Surface)

### 2.1 `server.ts` — Express API Server
| # | Severity | Issue | Location |
|---|----------|-------|----------|
| 1 | **Medium** | Gemini `apiKey` read at module top-level before `dotenv` loads — `process.env.GEMINI_API_KEY` may be `undefined` at import time | `server.ts:9` |
| 2 | **Medium** | `POST /api/generate-music` — no input validation on `mood`, `currentParams`, `settings` beyond existence checks | `server.ts:80-96` |
| 3 | **Low** | `catch (err)` swallows DB init failures silently — server continues without DB, no healthcheck | `server.ts:22` |
| 4 | **Medium** | `String(error)` in error responses leaks internal stack traces to clients in non-production | `server.ts:35,48,61,74,230` |
| 5 | **Low** | `POST /api/journals` returns a mock object when DB is missing, potentially misleading clients | `server.ts:41` |
| 6 | **Low** | Missing `pool.end()` on shutdown — DB connections leak on process exit | `db/index.ts:17-22` |
| 7 | **Medium** | No rate limiting on `/api/generate-music` — could be abused to consume Gemini API quota | server.ts:78 |

### 2.2 `src/App.tsx` — Main Component (760 lines)

## Phase 3: Test Suite

### 3.1 Unit Tests — `src/lib/synth.test.ts`
Test the `AmbientEngine` in isolation:
- `defaultParams` and `defaultSettings` match expected shapes and value ranges
- `applyEvolutionSettings()` updates `currentEvolutionSpeed` correctly
- `applyParams()` sets baseFreq, intervals, complexity without throwing
- `applyParams()` with boundary values (min/max for all numeric params)
- `updateTimbre()` (via `applyParams`) doesn't throw with any valid `AmbientParams`
- `start()` / `stop()` lifecycle transitions (mock Tone.js)
- `dispose()` calls stop + disposes all nodes without leaking

### 3.2 Unit Tests — `src/lib/validation.test.ts` (new utility)
Extract bounds-checking logic from `server.ts:192-218` into a pure function and test:
- `validateMusicParams()` clamps all 20+ numeric params to valid ranges
- `validateMusicParams()` defaults enums (`oscillatorType`, `noiseType`, `delayTime`) to valid values
- `validateMusicParams()` handles missing/malformed `colorPalette`
- `validateMusicParams()` handles missing/malformed `feedbackPrompt`

### 3.3 API Route Tests — `src/__tests__/api.test.ts`
Use `supertest` + Express app instance:
- `GET /api/journals` returns empty array when no DB

- `POST /api/interactions` returns 400 when `musicParams`/`userResponse` missing
- `POST /api/generate-music` returns 500 when GEMINI_API_KEY is missing
- `POST /api/generate-music` properly formats the prompt with journal/interaction context

### 3.4 Component Tests — `src/__tests__/App.test.tsx`
- Renders without crashing (smoke test)

### 3.5 Component Tests — `src/components/__tests__/AudioVisualizer.test.tsx`

### 3.6 Utility Tests — `src/components/__tests__/ParamSlider.test.tsx` (if extracted)
